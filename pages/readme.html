Introduction

My idea of is to create a simple blog that I want to host as GitHub Pages.

I want to write my text in Markdown, so I need some software to create the web site data.

The static site generator Pelican seems to be just what I need - lets give it a try.

All the Pelican project data are located in the branch main of my GitHub repository maro/nosense_pelican.

The created site data are stored in the branch gh-pages, to be served as GitHub Pages. I use the Python module ghp-import to copy the data to this branch.

Installation

The following Python modules are needed:

  • PyPi:pelican
  • PyPi:ghp-import
python3 -m venv --prompt pelican --without-pip ./venv
chmod 700 ./venv || exit 1
source ./venv/bin/activate || exit 1
#
python -m pip install --upgrade pip
python -m pip install --upgrade setuptools
python -m pip install --upgrade wheel
#
python -m pip install --upgrade "pelican[markdown]"
python -m pip install --upgrade ghp-import

To keep things simple, I use an own virtual environment for the installation.

Build/Test Commands

Build the Local Site

pelican

Run the Local Site

pelican --autoreload --listen

The local site is available at http://127.0.0.1:8000

Deploy the Site

pelican -s ./publishconf.py
ghp-import --no-jekyll --push --no-history ./output
